# Contributing to wista-project

## What You Can Contribute

- Bug reports and fixes
- Feature proposals
- Architectural ideas
- Documentation improvements
- Experiments and forks

Perfection is not required.
This project values exploration.

---

## Development Style

- Experimental changes are welcome
- Breaking changes are acceptable if clearly explained
- Forks are encouraged
- Clean separation of concerns is preferred

---

## Pull Requests

- Explain *why* the change exists
- Describe architectural impact if applicable
- Keep scope focused when possible

Draft PRs are welcome.

---

## Discussion & Philosophy

Architectural and philosophical discussions are welcome.
Please keep discussions constructive and respectful.

---

## Code of Conduct

All contributors are expected to follow the Code of Conduct.
